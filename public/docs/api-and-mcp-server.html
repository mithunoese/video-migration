02 — What is a REST API?

A REST API is a set of HTTP endpoints that let apps read and write data. You already use them — every time your browser fetches data from Supabase, it's hitting a REST API.

HTTP Methods
GET = read data
POST = create/send data
PUT = update data
DELETE = remove data
URL Patterns
GET /api/rates
GET /api/rates?from=USD&to=EUR
POST /api/convert
GET /api/history?days=30

Example request & response

// Browser or fetch() makes a request: GET /api/rates?from=USD&to=EUR // Your API responds with JSON: { "from": "USD", "to": "EUR", "rate": 0.92, "timestamp": "2026-02-17T12:00:00Z" }
For your OpenExchange demo: You already have exchange rate data. The API just wraps it in HTTP endpoints so any app (or MCP server) can access it programmatically. Built with Supabase Edge Functions — TypeScript running on Deno, deployed globally.
03 — What is an MCP Server?

MCP = Model Context Protocol. It's a standard that lets AI tools (Claude Code, Cursor, etc.) discover and call your functions. Instead of HTTP endpoints, you define "tools" that AI agents can use.

REST API
Defines endpoints
Called via HTTP requests
Consumed by apps, browsers, services
You decide what to call and when
MCP Server
Defines tools
Called via MCP protocol
Consumed by AI agents
The AI decides what to call and when

Registering a tool (TypeScript)

import { McpServer } from "@modelcontextprotocol/server"; import { z } from "zod/v4"; const server = new McpServer({ name: "openexchange", version: "1.0.0" }); server.registerTool("get_exchange_rate", { description: "Get current exchange rate between currencies", inputSchema: z.object({ from: z.string().describe("Source currency code e.g. USD"), to: z.string().describe("Target currency code e.g. EUR"), }), }, async ({ from, to }) => { // Call your API const res = await fetch(`https://your-api/rates?from=${from}&to=${to}`); const data = await res.json(); return { content: [{ type: "text", text: JSON.stringify(data) }] }; });

Transport Options

stdio (Local)
How your current MCP servers work
Runs as a subprocess on your machine. Claude Code spawns it automatically. This is how your GitHub, Context7, and Browserbase MCP servers run today. Best for: local development, personal tools.
Streamable HTTP (Remote)
Hosted web server anyone can connect to
Runs as an Express/HTTP server deployed to the cloud. Any MCP client can connect over the internet. Best for: shared tools, production, your portfolio visitors.

05 — The Build Plan
1
Define your API endpoints
Decide what data and actions to expose from OpenExchange
  • GET /rates — current exchange rates
  • POST /convert — convert amount between currencies
  • GET /history — historical rates over time
  • GET /currencies — list supported currencies
2
Build the API
Create Supabase Edge Functions for each endpoint
  • supabase functions new rates
  • Write TypeScript handler in supabase/functions/rates/index.ts
  • Connect to your Supabase Postgres for data
  • Deploy with supabase functions deploy
3
Build the MCP Server
TypeScript project using @modelcontextprotocol/server
  • npm init + install MCP SDK and Zod
  • Register tools: get_exchange_rate, convert_currency, get_rate_history
  • Each tool calls your API internally via fetch()
  • Use stdio transport for local dev
4
Connect to Claude Code
Add your MCP server to ~/.claude.json
  • Add to mcpServers config alongside GitHub, Context7, etc.
  • Claude Code auto-discovers your tools on startup
  • You can then say "convert 100 USD to EUR" and Claude uses your tool
5
Deploy & Connect to Portfolio
Make it live on your website
  • API endpoints already live on Supabase edge network
  • Frontend calls API from your portfolio site (React/Next.js)
  • Optional: deploy MCP server as HTTP for remote access
  • Optional: publish MCP server as npm package for others to use
