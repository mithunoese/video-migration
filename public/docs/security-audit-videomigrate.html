3 — Field Mapping Fixes

Problem

The Field Mapping table used native browser title attribute tooltips which overlapped between rows and truncated long field names. API field identifiers were invisible unless you hovered.

Fixes Applied

  • Replaced title tooltips with inline <code> labels — API field names now render directly below each friendly name
  • Added table-fixed layout with explicit column widths to prevent truncation
  • Expanded friendlyFieldName() dictionary with Zoom destination field names
  • Added whitespace-nowrap to status badges so they don't wrap mid-word
Before / After Comparison
Before: Hovering over “name” showed a tiny browser tooltip that overlapped with the row below. Zoom destination fields showed raw API keys like title with no friendly label.

After: Each field row shows the friendly name in bold plus the raw API identifier in a dimmed <code> block below it. No hover needed. Both source and destination fields have human-readable labels.

5 — Remaining Recommendations (Production)
  • Replace SSE token-in-URL with a short-lived ticket system to avoid JWT exposure in browser history and server logs
  • Bundle Tailwind at build time to eliminate unsafe-eval from the Content Security Policy
  • Move temp downloads from /tmp to a restricted directory with tighter permissions
  • Add file locking to cost_tracker.py JSON writes to prevent corruption under concurrent access
  • Store login lockout state in Redis for multi-worker deployments instead of in-process memory
  • Add rate limiting to all public-facing endpoints (consider slowapi or Nginx-level limits)
  • Enable HSTS by default when ENVIRONMENT != development
