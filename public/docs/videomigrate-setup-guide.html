1 AWS — Amazon Web Services
1

Sign in to AWS Console

Open console.aws.amazon.com and sign in. If you don't have an account yet, create a free-tier account at aws.amazon.com/free.

2

Open Security Credentials

Click your username in the top-right corner, then select "Security credentials" from the dropdown menu.

3

Create an Access Key

Scroll down to the "Access keys" section and click "Create access key".

4

Select CLI Use Case

Choose "Command Line Interface (CLI)" as the use case. Check the confirmation box at the bottom, then click Next.

5

Copy Both Keys

Click "Create access key". Copy both values and save them somewhere safe:

  • Access Key ID — starts with AKIA...
  • Secret Access Key — long random string

Important: You won't be able to see the Secret Access Key again after closing this page. Copy it now!

6

Configure AWS CLI

Open your terminal and run the configure command. Paste each value when prompted.

Terminal
aws configure
  • Access Key ID → paste your key
  • Secret Access Key → paste your secret
  • Default region → type us-east-1
  • Output format → type json
7

Create the S3 Staging Bucket

Create the bucket that will hold your video files during migration.

Terminal
aws s3 mb s3://oe-video-migration --region us-east-1
✓

Verify it worked: Run this command — you should see your Account ID, User ARN, and Account number.

aws sts get-caller-identity
2 KALTURA — VIDEO PLATFORM
1

Sign in to Kaltura Management Console

Open kmc.kaltura.com and sign in with your OpenExchange Kaltura credentials.

2

Go to Integration Settings

Click the Settings gear icon in the left sidebar, then click "Integration Settings".

3

Copy Your Credentials

On the Integration Settings page, you'll see:

  • Partner ID — a number like 1234567
  • Administrator Secret — a long alphanumeric string

Your User ID is the email you logged in with (e.g., mithun@openexchange.com).

4

Enter in Dashboard Settings

Go to the VideoMigrate dashboard → Settings tab. Paste your Partner ID, Admin Secret, and User ID into the Kaltura section.

✓

Verify it worked: Click "Test Connection" next to the Kaltura section in the Settings tab. You should see a green success indicator.

3 ZOOM — SERVER-TO-SERVER OAUTH
1

Open Zoom Marketplace

Go to marketplace.zoom.us and sign in with your Zoom admin account.

2

Create a Server-to-Server App

Click Develop in the top nav → Build App. Find "Server-to-Server OAuth" and click Create.

3

Name Your App

Enter VideoMigrate as the app name, then click Create.

4

Copy App Credentials

You'll land on the App Credentials page. Copy these three values:

  • Account ID
  • Client ID — this is what goes into "App ID" in the dashboard
  • Client Secret — this is what goes into "App Secret" in the dashboard
5

Add Required Scopes

Click Scopes in the left sidebar → "+ Add Scopes". Search for and add:

  • recording:write:admin (or recording:write)
  • recording:read:admin (or recording:read)

If using Zoom Events, also add event:write:admin.

6

Activate the App

Click Done to close the scopes dialog, then click the "Activate" button at the top of the page to make the app live.

7

Enter in Dashboard Settings

Go to the VideoMigrate dashboard → Settings tab. Paste your Account ID, Client ID (as App ID), and Client Secret (as App Secret) into the Zoom section.

✓

Verify it worked: Click "Test Connection" next to the Zoom section in the Settings tab. You should see a green success indicator.

4 CONNECT & TEST
1

Check Prerequisites

Go to Infrastructure tab → Click "Check Prerequisites". You should see 6/6 green checks.

2

Run Pipeline Test

Go to Settings tab → Click "Run Pipeline Test". This tests the full download → stage → upload flow with a single test video.

3

Start Migration

If the pipeline test passes, go to Migration tab → Start your first batch. Your videos will flow from Kaltura through S3 staging to Zoom.
