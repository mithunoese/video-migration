# Severity Location Vulnerability Recommended Fix
1 Critical app.py:72_hash_password()
Weak Password Hashing (SHA-256)
Uses hashlib.sha256() for password storage. SHA-256 is a cryptographic hash, not a password hash. It runs in nanoseconds, making brute-force and rainbow table attacks trivial. 		Replace with bcrypt or argon2. These are intentionally slow and include salting. Add bcrypt>=4.0.0 to requirements.txt.
2 Critical app.py:58JWT_SECRET init
JWT Secret Regenerates on Restart
If JWT_SECRET_KEY env var is unset, a random secret is generated each restart. On Vercel serverless, every cold start gets a different secret, silently invalidating all existing tokens. 		Require JWT_SECRET_KEY or derive a deterministic fallback from a stable project identifier. Log a warning at startup if using a generated secret.
3 Critical app.py:460/api/migration/stream
SSE Stream Unauthenticated
The Server-Sent Events endpoint for real-time migration progress has no auth check. Anyone can connect and receive all migration events including video IDs, titles, errors, and Zoom IDs. 		Add JWT token verification via query parameter (SSE can't use headers). Validate ?token=xxx before subscribing.
4 Critical app.py:507/api/migration/poll
Polling Endpoint Unauthenticated
The polling fallback for serverless environments also has no authentication. Exposes migration event history to unauthenticated users. 		Add Depends(_verify_jwt) to the endpoint.
5 Critical zoom_client.py:78_api_call() retry
Infinite Recursion on Rate Limit
When Zoom returns HTTP 429, _api_call() sleeps then calls itself recursively with no depth limit. Sustained rate limiting causes a stack overflow and crashes the process. 		Add a max_retries parameter (default 5). Track retry count and raise after exhaustion. Use iterative loop instead of recursion.

13 Medium index.html:7-9CDN script tags
CDN Scripts Without SRI
Tailwind CSS, Alpine.js, and Chart.js loaded from CDN without integrity (Subresource Integrity) attributes. A CDN compromise would allow arbitrary JavaScript execution. 		Add integrity="sha384-..." and crossorigin="anonymous" to each CDN script/link tag. Pin exact versions.
14 Medium app.py:224CSP header
CSP Allows unsafe-inline/unsafe-eval
Content Security Policy includes 'unsafe-inline' and 'unsafe-eval' for scripts. This significantly weakens XSS protection, as inline scripts and eval() remain executable. 		Move inline scripts to external files and use nonce-based CSP. Note: Alpine.js requires 'unsafe-eval' unless using the CSP build (@alpinejs/csp).
15 Medium app.py:262login endpoint
No Account Lockout
Rate limiting (10/min) is the only brute-force protection. No progressive lockout after repeated failures. An attacker can try 10 passwords every minute indefinitely. 		Track failed attempts per IP/username. Lock account for 15 min after 5 failures. Log and alert on lockout events.
16 Medium aws_staging.py:119 cost_tracker.py:39
State Files in /tmp (World-Readable)
Migration state and cost data stored in /tmp/migration-state.json and /tmp/video-migration/cost-data.json. On shared systems, /tmp is world-readable. Data includes video IDs, metadata, and spending. 		Use a configurable directory with restricted permissions (700). Default to ~/.video-migration/ instead of /tmp/.
17 Medium app.py:140-147Global state vars
Thread Safety Issues
Global mutable state (_migration_running, _sse_subscribers, _migration_events_store) is mutated from background threads without locks. Race conditions can cause missed events or inconsistent state. 		Add threading.Lock around all mutations of shared state. Use thread-safe collections where possible.

18 Low pipeline.py:106migrate_single_video()
Video ID Path Traversal
Entry ID used directly in file path construction: f"{entry_id}.mp4". If an entry_id contained ../, it could write outside the download directory. 		Sanitize entry_id: strip path separators, validate against [a-zA-Z0-9_-] pattern, or use Path.name to extract filename component only.
19 Low app.pyFastAPI config
No Request Body Size Limit
No maximum body size configured. An attacker could send arbitrarily large payloads to POST endpoints, consuming memory. 		Add body size middleware or configure behind a reverse proxy (nginx, Vercel) with size limits. Vercel has a 4.5MB default.
20 Low aws_staging.py:206,226DynamoDB scan()
DynamoDB Full Table Scans
get_pending_videos() and get_summary() perform full table scans. At scale (thousands of videos), this becomes slow and expensive. 		Add a Global Secondary Index (GSI) on the status field. Use query() instead of scan().
Next Steps Action Items
Priority 1: Fix Critical Issues (Immediate)
  • #1 Password Hashing: Replace hashlib.sha256 with bcrypt.hashpw(). Update .env.example password hash generation instructions.
  • #2 JWT Secret: Fail loudly or log a warning if JWT_SECRET_KEY is not set. For serverless, this env var is mandatory.
  • #3-4 Auth on SSE/Poll: Add token validation to /api/migration/stream (via query param) and /api/migration/poll (via header).
  • #5 Zoom Retry: Replace recursive retry with iterative loop, max 5 retries with exponential backoff.
Priority 2: Fix High Issues (Before Go-Live)
  • #6: Sanitize Claude API error in chat response.
  • #7-12: Add Depends(_verify_jwt) to all 6 unprotected endpoints: field-mapping PUT, costs/projection, costs/timeline, costs/export, report, test/result.
Priority 3: Medium/Low Issues (Next Sprint)
  • #13: Add SRI hashes to CDN scripts (generate via openssl dgst -sha384 -binary).
  • #14: Switch to Alpine.js CSP build or accept the tradeoff and document it.
  • #15: Implement progressive account lockout.
  • #16: Move state files out of /tmp.
  • #17: Add threading locks to global mutable state.
  • #18: Sanitize video IDs before path construction.
  • #19-20: Configure body size limits and DynamoDB GSI.
