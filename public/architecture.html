AWS — Staging Layer
Staging
AWS Infrastructure
Serverless orchestration with S3 for temporary storage and DynamoDB for migration state tracking.
S3 Bucket
Temporary video storage. Multipart upload for files >100MB. 5TB max per object. Lifecycle rules auto-delete after migration.
DynamoDB
State table per video: pending downloading staged uploaded failed. Prevents duplicates.
Step Functions
Distributed Map with MaxConcurrency: 100. No timeout limit. Built-in retry with exponential backoff.
Lambda
API calls + streaming. 15-min timeout per invocation. 10GB memory max.
15-min timeout for large files
EventBridge
Daily cron: 0 2 * * ?. Triggers Step Functions at 2 AM. Zero cost when idle.
Secrets Manager
Stores Kaltura Partner ID + Secret, Zoom OAuth credentials. Auto-rotation support.

Risk Points
Lambda 15-min Timeout
500MB files may exceed Lambda's 15-minute limit. Mitigation: Stream directly from Kaltura to S3 without buffering, or use Fargate tasks for large files.
Zoom h.264 Only
Zoom rejects h.265/HEVC files. Mitigation: Pre-check codec before upload, transcode with FFmpeg if needed.
Rate Limiting
Kaltura limits undocumented, Zoom caps at 30-80 req/sec. Mitigation: Cap concurrency at 100, implement exponential backoff.
Metadata Loss
Zoom lacks fields for tags, analytics, custom metadata. Mitigation: Append to description field, log unmapped data.
Zoom 2GB Upload Limit
Single-request uploads capped at 2GB. Mitigation: Use multipart upload endpoint for files exceeding 2GB.
Network Interruption
Partial downloads or uploads on flaky connections. Mitigation: Idempotent design with DynamoDB checkpointing. Resume from last state.
Build Options
Option A
Scripts + Cron
2-3 days to build
3 Python scripts: extract, stage, load. SQLite for state. Run on cron or GitHub Actions. Minimal infrastructure.
Best for POC
Option B
Step Functions
1-2 weeks to build
AWS Step Functions + Lambda + DynamoDB + CloudWatch. Serverless, auto-retry, built-in monitoring. Production-grade.
Production
Option C
Full Platform
3-4 weeks to build
Everything in B + web dashboard, per-client config, metadata mapping UI. Multi-tenant. Offer as a service.
Service Offering
Recommendation: Start with Option A. Run one real migration. Document edge cases. Then decide if the investment in B or C is justified by client demand.
